## 软著申请的痛点

最难的不是审批流程，而是**材料整理**：

| 材料项 | 难度 | 说明 |
|--------|------|------|
| 著作权登记申请表 | 中等 | 20多个字段：软件名称、版本号、开发环境等 |
| 操作说明书 | 较难 | 要像样，不能只有空泛功能列表 |
| 代码鉴别材料 | 较难 | 源代码前30页+后30页，共60页，格式严格 |
| 一致性校验 | 容易忽略 | 名称、版本号、页数在所有材料中必须一致 |

找代办：300-800元，且需把项目代码交给陌生人，**信息安全和时间成本双重损失**。

---

## 核心能力

1. **从真实源码抽取代码** — 100%来自你自己的项目，AI不编造任何一行源码
2. **自动处理前30页/后30页规则** — 不够60页自动按规则处理，无需手动数页数
3. **操作手册不套模板** — 先理解你的业务和功能，写面向审核员的说明
4. **申请表字段集中整理** — 全部统一生成到TXT，填报时对照复制
5. **每个关键节点都让你确认** — 业务口径、字段、代码、截图、草稿，AI不作主

```

    ├── 软件名称-代码(前30页).docx
    └── 软件名称-代码(后30页).docx
```

## 安装使用（三步）

```bash
# 第一步：下载代码
git clone https://github.com/Fokkyp/SoftwareCopyright-Skill.git
cd SoftwareCopyright-Skill

# 第二步：安装到Codex
mkdir -p ~/.codex/skills
cp -R software-copyright-materials ~/.codex/skills/

# 第三步：重启Codex，开始使用
# 打开你的项目目录，直接说：
使用 software-copyright-materials 生成当前项目的软件著作权申请资料
```

## 常见问题

**Q：这个工具真的完全免费吗？**
> 是的。MIT开源协议。不要相信任何用这个项目包装出来的付费服务。

**Q：需要什么环境？**
> 需要Codex和Python 3。可选安装 .NET SDK（增强DOCX生成）和 Chrome DevTools MCP（自动截图）。

**Q：生成的材料能直接提交吗？**
> 生成的是DOCX本地编辑稿，需先用Word/WPS检查，导出PDF后再按官网要求上传。申请表信息.txt是填报辅助，不是直接上传材料。

**Q：官网入口？**
> 中国版权保护中心 ccopyright.com.cn，进入著作权登记系统在线填报。
